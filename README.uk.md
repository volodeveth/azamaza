# Azamaza — Маркетплейс перевірених сервісів

Адаптивний хедер сайту з функціоналом пошуку, побудований на Next.js у рамках тестового завдання для фронтенд-розробника.

## Стек технологій

- **Фреймворк:** Next.js 16 (App Router)
- **Мова:** TypeScript
- **Стилізація:** Tailwind CSS v4
- **Шрифти:** Inter (основний), Onest (кнопки/мітки), Helvetica Neue (логотип)

## Архітектура

```
src/
├── app/
│   ├── layout.tsx          — Кореневий layout з підключенням шрифтів
│   ├── page.tsx            — Головна сторінка, композиція компонентів
│   ├── globals.css         — Дизайн-токени та глобальні стилі
│   └── api/search/
│       └── route.ts        — POST-ендпоінт для пошуку
├── components/
│   ├── Header/             — Логотип, навігація, бургер-меню
│   ├── Hero/               — Hero-секція з фоновим зображенням
│   ├── Categories/         — Динамічні кнопки категорій з мок-даних
│   ├── SearchForm/         — Форма: Destination, Service type, Date picker
│   │   ├── SearchForm.tsx          — Контейнер форми, валідація, логіка сабміту
│   │   ├── DestinationInput.tsx    — Текстовий інпут з кнопкою очищення
│   │   ├── ServiceTypeDropdown.tsx — Кастомний дропдаун з клавіатурною навігацією
│   │   └── DatePicker.tsx          — Нативний date picker з кастомним UI
│   ├── SearchResults/      — Відображення результатів після пошуку
│   └── ui/                 — Перевикористовувані UI-компоненти (Icons, ImageOverlay)
├── hooks/
│   ├── useClickOutside.ts  — Визначає кліки поза елементом
│   └── useSearch.ts        — Управління станом пошуку (loading/error/success)
├── data/
│   ├── categories.ts       — Мок-дані категорій та типів сервісів
│   └── searchResults.ts    — Мок-дані результатів пошуку
└── types/
    └── index.ts            — TypeScript інтерфейси
```

## Ключові рішення

- **Розділення компонентів:** Кожен UI-блок є ізольованим компонентом з власною відповідальністю (Header, Hero, Categories, SearchForm). Інпути форми додатково розділені на окремі компоненти для перевикористання.

- **Кастомні хуки:** `useSearch` інкапсулює логіку API-запитів та управління станом (loading/error/success). `useClickOutside` обробляє закриття дропдаунів/модалок — винесений як перевикористовуваний хук.

- **Контрольована форма:** Усі поля форми керуються через React state з валідацією кожного поля окремо. Дані форми передаються зверху вниз від SearchForm до дочірніх інпут-компонентів.

- **API роут:** `/api/search` — це Next.js Route Handler, який приймає POST-запити, валідує поля, імітує затримку мережі та повертає відфільтровані мок-дані.

- **Оверлеї зображень:** Меню, попап та сторінка категорій відображаються як оверлеї зображень (згідно з вимогами — ці блоки не потрібно верстати). Кожен оверлей підтримує закриття по кліку поза ним та клавішею Escape.

- **Mobile-first підхід:** Макет побудований для мобільних екранів 390px та масштабується до 768px (планшет). Десктоп-версія опціональна.

## Запуск

```bash
# Встановлення залежностей
npm install

# Запуск сервера розробки
npm run dev

# Збірка для продакшену
npm run build

# Запуск продакшен-сервера
npm start
```

Відкрити [http://localhost:3000](http://localhost:3000) у браузері.

## Адаптивні брейкпоінти

| Брейкпоінт | Опис |
|---|---|
| 390px | Мобільний — вертикальна форма, бургер-меню |
| 768px | Планшет — адаптовані відступи |
| 1280px+ | Десктоп (опціонально) |
